## Migration Guide

Follow these steps to upgrade smoothly, especially if you encounter breaking changes.

<Steps>
  <Step title="Backup Your Site" icon="database">
    Create a full backup:

    ```bash
    git clone your-repo backup-asurad-v1.1
    ```
  </Step>
  <Step title="Update Dependencies" icon="package">
    <Tabs>
      <Tab title="npm" icon="npm">
        ```bash
        npm update asurad-docs@latest
        ```
      </Tab>
      <Tab title="yarn" icon="yarn">
        ```bash
        yarn upgrade asurad-docs@latest
        ```
      </Tab>
    </Tabs>
  </Step>
  <Step title="Test Components" icon="test-tube">
    Verify key components like `<Update>` and Mermaid diagrams:

    ```mermaid
    graph TD
        A[Current Version] --> B[Upgrade]
        B --> C[New Features]
        C --> D[Production]
    ```
  </Step>
  <Step title="Deploy" icon="rocket">
    Push changes and monitor for issues.
  </Step>
</Steps>

<Expandable title="Detailed Breaking Changes" default-open="false">

### v1.1.0 Changes

- Remove all `<Mermaid>` JSX tags; use code blocks instead
- `{API_KEY}` now requires backtick escaping in prose: `` `API_KEY` ``

| Change | Impact | Fix |
|--------|--------|-----|
| No `<Mermaid>` component | Parsing errors | Use ```` ```mermaid ```` blocks |
| Stricter frontmatter | YAML failures | No blank lines inside `---` |

</Expandable>
